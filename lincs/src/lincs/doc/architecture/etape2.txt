> -une librairie d’emulation qui intercepte les appels open et ouvre à la 
> place un «pipe» pour exécuter un utilitaire, cscat. Pour le moment, ceci 
> semble suffisant

En effet, il est mieux de conserver les adaptateurs pour les divers
formats d'anciens fichiers /etc séparés dans des modules autonomes.
Chaque module pourrait être un exécutable à executer avec popen par la
librairie qui intercepte "open"; ce pourrait aussi être un fichier .so
chargé par cscat, il faudrait alors que cscat recoive en argument le nom
de fichier /etc à générer qui identifierait le fichier .so à charger.
Le plus simple est probablement un simple script exécuté par popen et
dont le nom correspond à celui du fichier de /etc à générer, par exemple
/etc/confi/cat/modules.conf pour générer le contenu de
/etc/modules.conf.

> -un seul module backend qui garde les valeurs dans des fichiers

Tel que nous en avions discuté, chaque couche doit être conservée
séparément pour facilement pouvoir être copiée d'un système à l'autre.

> -la structure pour gérer les valeurs devrait être complètement 
> retravaillée, justement pour supporter les schémas et les couches 
> pleinement. Probablement aussi la notions de metadonnees, a la gconf a 
> beaucoup de sens. Aussi, peut-être une liste d’attribut nom-valeur 
> remplacerait le contenu proprement dit de la valeur, pour plus de génialité 
> (cela semble se chevaucher un peu avec les autres composantes de la 
> structure).

Une seule valeur par "nom" avec une liste d'attributs extensibles peut
être une bonne solution; cela permet de s'adapter aux besoins de
supporter gconf, d'associer plus de description dans les template
incluant des informations pour l'affichage dans l'éditeur (icône,
contraintes...). Ainsi la méta information serait extensible, un peu
comme les extended attributes de ext2. Par contre si on a plusieurs
valeurs, c'est mieux d'avoir un répertoire qui contient ces valeurs.

avec des listes d'accès:

- pour les accès locaux, le uid peut être authentifié facilement,
- pour les accès distants, les certificats peuvent être vérifiés avec
